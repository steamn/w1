По маршруту /fetch/ должна располагаться (выдаваться клиенту) веб-страница. К ней предъявляются следующие требования: А. Должна содержать одно поле ввода input с идентификатором (id) inp Б. Должна содержать одну кнопку button с идентификатором (id) bt В. При щелчке по этой кнопке сценарий на этой странице должен с помощью fetch обратиться к адресу, который к моменту щелчка введён в поле ввода (адрес будет вводить автоматический тестер) Г. Получив результат, возвращённый этим адресом, сценарий должен поместить этот результат в поле inp вместо того значения, что в нём было ранее. Примечание 1. При использовании метода fetch нужно преобразовать Response с помощью метода text().
